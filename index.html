查看docker网络接口和容器网络接口的关系

  • docker本机的网络接口：docker0

  • 容器的网络接口：vethf717ba5

  • 查看docker0被哪些容器的网络接口连接

none

  • 没有ip和mac地址
  • 适用于安全性要求高的容器

host

  • 与主机的网络共享
  • 安全性低
  • 存在主机端口和容器端口冲突的情况

本机容器间的连通

--link

作用：使container-1连通container-2（注意–link是单向连通，即container-1能通container-2，反过来不通）

--network

作用：使container-1和container-2互相连通

--network connect

作用：使container连接到指定的网络

小结

  • 两个容器如果连接到自定义的网络（不是docker0的网络），默认可以使用网络接口名称访问。

  • 两个容器如果连接到docker0，必须使用--link才能使用网络接口名称访问。

Nginx-进程优化

发表于 2020-08-14 分类于 Nginx

优化Nginx的CPU利用率

Nginx进程间不做无用功浪费CPU资源

  • worker进程不应该在繁忙时，主动让出CPU。
    • worker进程间竞争CPU造成资源耗散。
      • worker进程数 = CPU核数
    • worker进程不应调用一些阻塞API导致主动让出CPU。
      • 谨慎考虑使用阻塞的第三方API调用

防止其他进程竞争CPU资源

  • 提升优先级占用CPU更长的时间
  • 减少操作系统上耗资源的非Nginx进程

阻塞API导致主动让出CPU的场景

  • 速度不一致引发的阻塞API
    • 硬件执行速度不一致，例如CPU和磁盘
  • 业务场景产生的阻塞API
    • 例如同步读网络报文

减少进程间切换

Nginx worker尽可能的处于R状态

  • uptime/top：查看负载

尽可能的减少进程间切换

  • 减少主动切换
  • 减少被动切换
    • 增大进程优先级
  • 绑定CPU

延迟处理新连接

查看上下文切换次数

  • vmstat
    • cs
  • dstat
    • csw
  • pidstat -w -p
    • cswch/s：主动切换
    • nvcswch/s：被动切换

决定CPU时间片的大小

Nice静态优先级

  • -20 ~ 19：时间片大 -> 小

Priority动态优先级

  • 0 ~ 139：由系统分析之后动态调整，用户不能直接修改

设置worker进程的静态优先级

进程间负载均衡

accept_mutex

  • on：linux内核2.6版本以前，开启accept_mutex锁解决accept的惊群效应。
  • **off(reuseport)**：linux内核2.6版本以后，内核解决了accept的惊群效应。

多队列网卡对多核CPU的优化

  • RSS：Receive Side Scaling
    • 硬中断负载均衡：每个CPU都有网卡队列，CPU可以并发处理硬中断。
  • RPS：Receive Packet Steering
    • 软中断负载均衡
  • RFS：Receive Flow Steering

empty_gif模块

模块

  • ngx_http_empty_gif_module模块，默认编译，–without-http_empty_gif_module禁用

功能

  • 从前端页面做用户行为分析时，由于跨域等要求，前端打点的上报数据一般是GET请求，且考虑到浏览器解析DOM树的性能消耗，所以请求透明图片消耗最小，而1*1的gif图片体积最小（仅43字节），故通常请求gif图片，并在请求中把用户行为信息上报服务器。
  • Nginx可以在access日志中获取到请求参数，进而统计用户行为。但若在磁盘中读取1*1的文件则有磁盘IO消耗，empty_gif模块将图片放在内存中，加快了处理速度。

access日志压缩

  • buffer默认64KB
  • gzip默认级别为1
  • 通过zcat解压查看日志

error日志输出内存

场景

  • 在开发环境下定位问题时，如果打开了debug级别日志，大量的debug日志会引发性能问题，可以将日志输出到内存中。

配置语法

  • error_log memory:32m debug;

查看内存中日志的方法

  • gdb -p [worker进程pid] -ex "source nginx.gdb" --batch
