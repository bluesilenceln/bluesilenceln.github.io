发表于 2020-02-26 更新于 2020-08-23 分类于 Nginx

优化Nginx的CPU利用率

Nginx进程间不做无用功浪费CPU资源

  • worker进程不应该在繁忙时，主动让出CPU。
    • worker进程间竞争CPU造成资源耗散。
      • worker进程数 = CPU核数
    • worker进程不应调用一些阻塞API导致主动让出CPU。
      • 谨慎考虑使用阻塞的第三方API调用

防止其他进程竞争CPU资源

  • 提升优先级占用CPU更长的时间
  • 减少操作系统上耗资源的非Nginx进程

阻塞API导致主动让出CPU的场景

  • 速度不一致引发的阻塞API
    • 硬件执行速度不一致，例如CPU和磁盘
  • 业务场景产生的阻塞API
    • 例如同步读网络报文

  • vmstat
    • cs
  • dstat
    • csw
  • pidstat -w -p
    • cswch/s：主动切换
    • nvcswch/s：被动切换

决定CPU时间片的大小

Nice静态优先级

  • -20 ~ 19：时间片大 -> 小

Priority动态优先级

  • 0 ~ 139：由系统分析之后动态调整，用户不能直接修改

进程间负载均衡

accept_mutex

  • on：linux内核2.6版本以前，开启accept_mutex锁解决accept的惊群效应。
  • **off(reuseport)**：linux内核2.6版本以后，内核解决了accept的惊群效应。

多队列网卡对多核CPU的优化

  • RSS：Receive Side Scaling
    • 硬中断负载均衡：每个CPU都有网卡队列，CPU可以并发处理硬中断。
  • RPS：Receive Packet Steering
    • 软中断负载均衡
  • RFS：Receive Flow Steering
